克隆http方式记住帐号密码

$ git config --global user.name "wangliang"

$ git config --global user.email wangliang@51Yjtc.com

用来注册用户和邮箱 这会在C:\Users\Administrator下多出一个.gitconfig

在home文件夹，一般是 C:\Documents and Settings\Administrator 下建立文件 .git-credentials
（windows下不允许直接创建以.开头的文件，所以有一个小技巧：先创建一个文件名叫 ）git-credentials 然后进入 git bash 使用命令：
mv git-credentials .git-credentials
用记事本打开这文件输入：
https://{username}:{password}@github.com
比如：
http://wangliang:123456@gitlab.51yjtc.com
保存
添加config项
在任意文件夹下右键进入 git bash
然后输入：
git config --global credential.helper store
执行完后去查看 C:\Documents and Settings\Administrator\.gitconfig 这个文件，发现多了一项：
[credential]
helper = store
就成功了。
然后要重开 git bash 窗口，再提交就不用输入用户名密码

克隆ssh方式记住帐号密码

等待git和Tortoisegit都安装完成后，按照以下步骤走

1、打开TortoiseGit下的PuttyGen，在打开的窗口中点击Generate按钮，会出现绿色的进度条，等待生成，生成的过程中你需要将鼠标放在绿色进度条上晃动，以便进度条前进

2、生成之后复制当前生成的key，窗口不要关闭

3、登录你的githup，点击右上方的小人物图标(Profile settings),进去设置页面之后，选择左侧选项中的SSH Keys,点击Add SSH Key按钮，把你刚才复制的key粘贴到key文本框中，然后点击Add key按钮即可

4、返回到生成key的窗口，点击Save private key按钮保存为适用于TortoiseGit的私钥扩展名为.ppk,保存到哪个磁盘下都可以

5、打开TortoiseGit下的settings，找到git下的Remote，将Putty的路径设置成你保存私钥扩展名.ppk保存的路径即可

5、需要在你的本地磁盘(我的在E盘)中建立一个文件夹，以便需要克隆的文件克隆到当前文件夹内(我的是文件夹名称是：git)

git常用命令

添加修改：git add -u

提交注释修改：git commit -m "修改了css"

更新：git pull 或 git pull origin develop

提交：git push 或 git push origin develop

查看本地分支：git branch

查看所有分支：git branch -a

切换分支：git checkout develop

新建本地分支：git checkout -b newfenzhi

合并分支：git merge
